# 页面相关数据库设计v6.0

> 说明：本版本不再沿用“固定三级页面结构”作为核心模型，而是升级为：
>
> **页面（page） → 页面版本（page_version） → 页面节点树（page_node）**
>
> 并配套：
>
> - 组件注册中心（component_def）
> - 数据绑定（data_binding）
> - 页面模板（page_template）
> - 可复用片段（reusable_fragment）
>
> 这样页面不再被限制为 `page → section → block` 三层，而是支持受控递归、多层容器、插槽 slot、模板复用、片段复用、独立数据源绑定与后续动作扩展。

---

# 一、整体设计思路

## 1.1 原有三级版的问题

旧版模型核心为：

**page → page_section → page_block**

这种结构适合官网类页面的基础搭建，但本质上仍然是固定层级模型，主要局限在于：

1. 页面层级写死，难以支撑真正多层结构；
2. `block_type` 只是枚举值，不是真正的组件注册中心；
3. `props_json` 承担过多职责，结构配置、业务配置、样式配置混在一起；
4. 数据源绑定过于轻量，难以支持统一的数据驱动渲染；
5. 模板、复用、版本、动作没有形成系统能力。

## 1.2 新版设计目标

本版的核心目标是把数据库升级为**平台型低代码模型**，具备以下能力：

- 页面结构树化，而不是固定三级；
- 组件注册化，而不是只靠 `block_type`；
- 数据绑定独立化，而不是只靠 `data_source_type + data_source_value`；
- 模板与可复用片段独立建模；
- 页面编辑与页面发布分离；
- 后续可扩展事件、动作、显示规则与 schema 驱动属性编辑。

## 1.3 新版核心结构

新版的页面搭建主线为：

**menu → page → page_version → page_node**

配套结构为：

- `component_def`：组件注册中心
- `data_binding`：数据绑定
- `page_template`：页面模板
- `reusable_fragment`：可复用片段

页面真正的内容结构，不再靠 `page_section / page_block`，而是由 `page_node` 这一棵树承载。

# 二、表结构总览

本版数据库核心表如下：

## 2.1 页面与导航主线

- `menu`：导航菜单
- `page`：页面元信息
- `page_version`：页面版本

## 2.2 页面结构主线

- `component_def`：组件定义
- `page_node`：页面节点树
- `data_binding`：数据绑定

## 2.3 模板与复用

- `page_template`：页面模板
- `reusable_fragment`：可复用片段

## 2.4 内容数据源

- `news_category`
- `news`
- `banner`
- `site_config`

> 注：旧版 `page_section / page_block` 不再作为核心表使用。
> 如需兼容旧数据，可保留为 legacy 表，在迁移完成后逐步废弃。

# 三、表结构说明

## 1. 菜单表（menu）

用于管理网站前台导航菜单。

| 字段名       | 数据类型                | 约束                  | 描述                                       |
| ------------ | ----------------------- | --------------------- | ------------------------------------------ |
| id           | bigint                  | PRIMARY KEY           | 菜单唯一标识                               |
| parent_id    | bigint                  | FOREIGN KEY (menu.id) | 父菜单ID，顶级菜单可为 NULL                |
| name         | varchar(100)            | NOT NULL              | 菜单显示名称                               |
| code         | varchar(100)            | UNIQUE                | 菜单编码                                   |
| url_type     | enum('page','external') | NOT NULL              | 链接类型：page-站内页面；external-外部链接 |
| page_id      | bigint                  | FOREIGN KEY (page.id) | 当 `url_type=page` 时，关联页面 ID         |
| external_url | varchar(500)            |                       | 当 `url_type=external` 时，存储完整 URL    |
| sort_order   | int                     |                       | 同级菜单排序                               |
| status       | boolean                 | NOT NULL              | 状态：true-显示；false-隐藏                |
| icon         | varchar(100)            |                       | 图标（预留）                               |
| remark       | varchar(255)            |                       | 备注                                       |
| created_at   | datetime                |                       | 记录创建时间                               |
| updated_at   | datetime                |                       | 记录最后更新时间                           |

### 说明

- 本表继续沿用旧设计；
- 菜单层仍然只负责导航入口，不承担页面内容结构；
- 真正的页面内容由 `page_version + page_node` 负责。

## 2. 页面表（page）

用于定义系统中有哪些页面。

| 字段名             | 数据类型     | 约束            | 描述                        |
| ------------------ | ------------ | --------------- | --------------------------- |
| id                 | bigint       | PRIMARY KEY     | 页面唯一标识                |
| name               | varchar(100) | NOT NULL        | 页面名称（后台管理用）      |
| code               | varchar(100) | UNIQUE NOT NULL | 页面编码                    |
| path               | varchar(200) | UNIQUE NOT NULL | 前台访问路径                |
| page_type          | varchar(50)  | NOT NULL        | 页面类型                    |
| title              | varchar(200) |                 | 页面标题                    |
| status             | boolean      | NOT NULL        | 状态：true-启用；false-停用 |
| current_version_id | bigint       | FOREIGN KEY     | 当前发布版本 ID             |
| seo_title          | varchar(200) |                 | SEO 标题                    |
| seo_keywords       | varchar(200) |                 | SEO 关键词                  |
| seo_description    | text         |                 | SEO 描述                    |
| remark             | varchar(255) |                 | 备注                        |
| created_at         | datetime     |                 | 记录创建时间                |
| updated_at         | datetime     |                 | 记录最后更新时间            |

### page_type 建议值

- `portal`：门户页
- `single`：单页
- `service`：服务页
- `channel`：频道页
- `custom`：自定义页

### 说明

- `page` 只保存页面元信息；
- 页面真正的编辑内容不直接挂在 `page` 上，而是挂在 `page_version` 上；
- `current_version_id` 表示当前线上使用的版本。

## 3. 页面版本表（page_version）

用于实现草稿版、发布版、历史版管理。

| 字段名       | 数据类型     | 约束                  | 描述                         |
| ------------ | ------------ | --------------------- | ---------------------------- |
| id           | bigint       | PRIMARY KEY           | 版本唯一标识                 |
| page_id      | bigint       | FOREIGN KEY (page.id) | 所属页面 ID                  |
| version_no   | int          | NOT NULL              | 版本号                       |
| version_name | varchar(100) |                       | 版本名称                     |
| source_type  | varchar(50)  |                       | 来源类型                     |
| source_id    | bigint       |                       | 来源模板/来源版本 ID         |
| status       | varchar(30)  | NOT NULL              | draft / published / archived |
| is_locked    | boolean      | NOT NULL              | 是否锁定                     |
| remark       | varchar(255) |                       | 备注                         |
| created_by   | varchar(100) |                       | 创建人                       |
| created_at   | datetime     |                       | 创建时间                     |
| updated_at   | datetime     |                       | 更新时间                     |

### source_type 建议值

- `manual`
- `template`
- `clone_version`

### 说明

- 页面编辑永远作用在某个版本上；
- 发布时，`page.current_version_id` 指向某个 `page_version.id`；
- 这样页面内容编辑与页面线上展示彻底分离。

## 8. 页面节点表（page_node）

> 本表是新版数据库的核心，负责承载页面结构树。

| 字段名            | 数据类型     | 约束                       | 描述                                 |
| ----------------- | ------------ | -------------------------- | ------------------------------------ |
| id                | bigint       | PRIMARY KEY                | 节点唯一标识                         |
| page_version_id   | bigint       | FOREIGN KEY                | 所属页面版本 ID，可空                |
| template_id       | bigint       | FOREIGN KEY                | 所属页面模板 ID，可空                |
| fragment_id       | bigint       | FOREIGN KEY                | 所属复用片段 ID，可空                |
| parent_id         | bigint       | FOREIGN KEY (page_node.id) | 父节点 ID，顶层节点可为 NULL         |
| node_type         | varchar(50)  | NOT NULL                   | 节点类型                             |
| component_key     | varchar(100) |                            | 对应组件 key                         |
| node_name         | varchar(100) |                            | 节点名称                             |
| slot_name         | varchar(100) |                            | 插槽名 / 区域名                      |
| sort_order        | int          |                            | 同层级排序                           |
| depth             | int          |                            | 节点深度                             |
| col_span          | int          |                            | 栅格列占比                           |
| row_span          | int          |                            | 栅格行占比                           |
| data_binding_id   | bigint       | FOREIGN KEY                | 数据绑定 ID                          |
| ref_fragment_id   | bigint       | FOREIGN KEY                | 当节点引用某个可复用片段时，指向片段 |
| props_json        | json         |                            | 业务属性配置                         |
| style_json        | json         |                            | 样式配置                             |
| layout_json       | json         |                            | 布局配置                             |
| event_json        | json         |                            | 事件/动作配置                        |
| visible_rule_json | json         |                            | 显示规则                             |
| status            | boolean      | NOT NULL                   | 状态：true-启用；false-停用          |
| remark            | varchar(255) |                            | 备注                                 |
| created_at        | datetime     |                            | 创建时间                             |
| updated_at        | datetime     |                            | 更新时间                             |

### node_type 建议值

- `container`
- `component`
- `fragment_ref`

### 结构归属规则

`page_node` 中以下三个字段必须**三选一**：

- `page_version_id`
- `template_id`
- `fragment_id`

即：

- 页面节点属于某个页面版本；
- 或属于某个页面模板；
- 或属于某个可复用片段。

### slot_name 示例

- `main`
- `left`
- `right`
- `header`
- `footer`
- `tab_1`
- `tab_2`

### props_json 示例

```json
{
  "title": "通知公告",
  "count": 5,
  "showDate": true,
  "showMore": true
}
```

### style_json 示例

```json
{
  "backgroundColor": "#ffffff",
  "padding": 16,
  "borderRadius": 12
}
```

### layout_json 示例

```json
{
  "layoutType": "grid",
  "columns": 2,
  "gap": 16,
  "align": "stretch"
}
```

### event_json 示例

```json
{
  "onClick": {
    "action": "navigate",
    "target": "/news"
  }
}
```

### visible_rule_json 示例

```json
{
  "showOn": ["pc", "mobile"],
  "loginRequired": false
}
```

### 说明

- 这是新版最关键的一张表；
- 页面结构不再依赖固定的 section/block 层级；
- 页面中的大区域、栅格、标签页、普通内容组件，本质上都只是 `page_node` 的不同实例；
- `slot_name` 用于支持插槽式布局；
- `ref_fragment_id` 用于在页面中插入一个可复用片段引用节点。

# 四、关系说明

## 4.1 页面发布链路

- `page` 定义页面本体；
- `page_version` 维护该页面的多个版本；
- `page.current_version_id` 指向当前线上版本；
- `page_node` 负责存储该版本下的页面结构树。

## 4.2 模板生成链路

- `page_template` 定义模板；
- 模板结构存放于 `page_node.template_id`；
- 创建页面时，从模板生成 `page_version` 与节点树。

## 4.3 片段复用链路

- `reusable_fragment` 定义可复用片段；
- 片段结构存放于 `page_node.fragment_id`；
- 页面节点中可通过 `node_type = fragment_ref + ref_fragment_id` 引用该片段。

## 4.4 组件注册链路

- 所有可选组件在 `component_def` 中定义；
- 页面节点通过 `component_key` 绑定到某个组件定义；
- 后台属性面板可由 schema 动态生成。

## 4.5 数据绑定链路

- `data_binding` 统一定义绑定方案；
- 页面节点通过 `data_binding_id` 引用绑定方案；
- 节点渲染时，先取绑定数据，再映射到组件显示层。

# 五、典型数据组织示例

## 5.1 首页结构示例

### 页面本体

| id   | name | code | path | page_type | current_version_id |
| ---- | ---- | ---- | ---- | --------- | ------------------ |
| 1    | 首页 | home | /    | portal    | 101                |

### 页面版本

| id   | page_id | version_no | version_name | status    |
| ---- | ------- | ---------- | ------------ | --------- |
| 101  | 1       | 1          | 首页正式版   | published |

### 页面节点

| id   | page_version_id | parent_id | node_type | component_key        | node_name  | slot_name | sort_order |
| ---- | --------------- | --------- | --------- | -------------------- | ---------- | --------- | ---------- |
| 1    | 101             | NULL      | container | page_container       | 页面根容器 | main      | 1          |
| 2    | 101             | 1         | container | hero_split_container | 首屏容器   | main      | 1          |
| 3    | 101             | 2         | component | hero_banner          | 首页轮播   | left      | 1          |
| 4    | 101             | 2         | component | notice_list          | 通知公告   | right     | 2          |
| 5    | 101             | 1         | container | grid_container       | 新闻区容器 | main      | 2          |
| 6    | 101             | 5         | component | news_list            | 学会新闻   | left      | 1          |
| 7    | 101             | 5         | component | quick_links          | 快捷入口   | right     | 2          |

可以看到，这里已经不是固定三级，而是：

**page → version → node(root) → node(container) → node(component)**

且继续向下嵌套也没有结构性障碍。

# 七、落地建议

## 7.1 后台前端编辑器如何对应

这版数据库落地到后台前端时，可直接对应成：

- 左侧：节点树（page_node）
- 中间：页面预览（按节点树递归渲染）
- 右侧：属性面板（按 component_def.schema 动态渲染）

## 7.2 版本管理建议

建议后台只允许：

- 编辑 draft 版本；
- 发布时切换 `page.current_version_id`；
- 线上页面只读当前发布版本，不直接读最新草稿。

## 7.3 模板与片段使用建议

- 创建页面：优先从 `page_template` 初始化；
- 跨页面公共区块：优先沉淀到 `reusable_fragment`；
- 页面中引用公共片段：通过 `page_node.ref_fragment_id`。

# 八、结论

本版数据库设计相较旧版的根本变化，不是“从三级改成四级”，而是：

**从固定层级模型，改成节点树模型。**

它具备以下平台级特征：

1. 页面结构递归化；
2. 组件注册化；
3. 数据绑定独立化；
4. 模板独立化；
5. 片段复用化；
6. 版本发布化；
7. 事件动作预留化；
8. 可支撑 schema 驱动后台编辑器。

这才是更接近优秀低代码平台共同特征的数据库结构。
