# 前端需求接口文档 v2.0

## 1. 文档说明

本文档用于明确低代码后台前端对后端 REST 接口的需求，供后端实现与前后端联调。

**v2.0 修订要点**：字段名、请求体与筛选参数与根目录《数据库设计_v6.0_平台型低代码版.md》（以下简称 **v6.0**）中的 **`menu`、`page`、`page_version`、`page_template`、`reusable_fragment`、`component_def`、`data_binding`、`page_node`** 表保持一致；并不扩展 v6.0 未定义的业务字段。

本文档描述当前前端已使用或已登记的接口能力（含 `src/api/*ApiDrafts` 中的草案）。**v6.0 中的内容类表**（如 `news`、`banner`、`site_config` 等）不在本文范围内。

编辑器相关接口重点约定：节点树平铺读取、结构移动与内容更新分离、整树全量保存语义。

## 2. 统一约定

### 2.1 基本约定

- 接口风格：REST 为主
- 主键类型：`number`（与 bigint 对应）
- 布尔状态字段：`boolean`
- 时间字段：ISO 8601 字符串
- JSON 列（v6.0 中为 `json`）：传输层使用 **JSON 字符串**（字段名 `*_json`），内容为合法 JSON 文本

### 2.2 响应包装（推荐）

后端可采用统一信封，例如：

```ts
{
  success?: boolean
  code?: string | number
  message?: string
  data?: T
}
```

**业务数据**位于 `data`，或与信封并列时以前端约定为准。列表、详情、节点树等均可能出现在 `data` 内。

### 2.3 列表分页与列表载荷

**约定形态 A（推荐，与现有前端列表一致）**：

```ts
{
  items: T[],
  total: number,
  page?: number,
  page_size?: number
}
```

**兼容形态 B（部分集成接口已出现）**：`data.records` + `data.total`，分页字段 `current` / `size`（或 `page` / `page_size`）。后端若采用形态 B，应在联调说明中固定字段名。

详情接口：`data` 内为实体 `T`，或直接返回 `T`（需前后端统一）。

删除接口：`{ id: number }`（可在 `data` 内）。

节点树保存成功：`{ nodes: PageNode[] }`（可在 `data` 内）。

### 2.4 通用约束（编辑器）

- 节点树读取接口统一返回**平铺数组 `nodes[]`**，不返回嵌套树
- **结构字段**：`parent_id`、`slot_name`、`sort_order`（及移动接口）
- **内容字段**：`node_name`、`component_key`、绑定引用、`props_json`、`style_json`、`layout_json`、`event_json`、`visible_rule_json` 等
- 结构调整统一走 **`POST /page-nodes/{id}/move`**，不得通过节点更新接口改结构字段

### 2.5 `page_node` 归属（与 v6.0 一致）

每条节点在持久化后 **`page_version_id`、`template_id`、`fragment_id` 三者有且仅有一个非空**（由上下文接口隐含或写入时校验）。

## 5. 页面管理接口

### 5.1 页面列表

- `GET /pages`

请求参数：

```ts
{
  name?: string
  code?: string
  path?: string
  page_type?: string
  title?: string
  status?: boolean
  current_version_id?: number
  page?: number
  page_size?: number
}
```

返回：列表载荷（§2.3），元素为 **Page**。

### 5.2 页面详情

- `GET /pages/{id}` → **Page**

### 5.3 当前发布版本

- `GET /pages/{id}/current-version`

返回：**PageVersion**，或与发布流程约定一致；若无当前版本可为 `null` 或 `404`（联调确定）。

### 5.4 新建页面

- `POST /pages`

请求体（与 v6.0 `page` 可写列一致，不含 `current_version_id`）：

```ts
{
  name: string
  code: string
  path: string
  page_type: string
  title: string | null
  status: boolean
  seo_title: string | null
  seo_keywords: string | null
  seo_description: string | null
  remark: string | null
}
```

返回：**Page**

### 5.5 更新页面

- `PUT /pages/{id}`，请求体同 §5.4 → **Page**

### 5.6 删除页面

- `DELETE /pages/{id}` → `{ id: number }`

## 6. 页面版本接口

### 6.1 版本列表

- `GET /pages/{pageId}/versions`

请求参数：

```ts
{
  status?: 'draft' | 'published' | 'archived'
  version_name?: string
  source_type?: string
  is_locked?: boolean
  page?: number
  page_size?: number
}
```

返回：列表载荷，元素为 **PageVersion**。

### 6.2 版本详情

- `GET /page-versions/{id}` → **PageVersion**

### 6.3 新建版本

- `POST /pages/{pageId}/versions`

请求体（与 v6.0 写入语义一致；`version_no` 由服务端生成）：

```ts
{
  version_name: string | null
  source_type: 'manual' | 'clone_version' | null
  source_id: number | null
  remark: string | null
}
```

说明：

- `source_type = 'clone_version'` 时，`source_id` 为被复制的 **page_version.id**
- 从模板创建**不使用本接口**，使用 §6.4

返回：**PageVersion**

### 6.4 从模板创建版本

- `POST /pages/{pageId}/versions/from-template`

**推荐**请求体：`Content-Type: application/json`

```ts
{
  template_id: number
  version_name: string | null
  remark: string | null
}
```

返回：**PageVersion**（创建后应已复制模板 `page_node` 树至新版本）。

> **兼容说明**：个别集成环境曾使用 `application/x-www-form-urlencoded` 及驼峰字段名，应在联调中单端固化；新接入建议统一 JSON + snake_case，与全文约定一致。

### 6.5 发布版本

- `POST /page-versions/{id}/publish`

返回至少包含：

```ts
{
  page_version: PageVersion
  page: Pick<Page, 'id' | 'current_version_id'> | Page
}
```

即至少更新并返回 **`page.current_version_id`** 与对应 **PageVersion**；是否返回完整 **Page** 由后端决定。

### 6.6 锁定 / 解锁版本

- `POST /page-versions/{id}/lock`
- `POST /page-versions/{id}/unlock`

返回：**PageVersion**

### 6.7 复制版本

- `POST /page-versions/{id}/clone`

请求体：

```ts
{
  version_name?: string | null
  remark?: string | null
}
```

返回：**PageVersion**（含复制后的节点树，后端保证）。

### 6.8 删除版本

- `DELETE /page-versions/{id}` → `{ id: number }`

```ts
  url_type?: 'page' | 'external'
  status?: boolean
  page?: number
  page_size?: number
}
```

返回：列表载荷，元素为 **Menu**（**平铺**；前端自行组树）。

### 11.2 菜单详情

- `GET /menus/{id}` → **Menu**

### 11.3 新建菜单

请求体：

```ts
{
  parent_id: number | null

  code: string
  url_type: 'page' | 'external'
  page_id: number | null
  external_url: string | null
  sort_order: number | null
  status: boolean
  icon: string | null
  remark: string | null

```

返回：**Menu**

### 11.4 更新菜单

- `PUT /menus/{id}`，请求体同 §11.3 → **Menu**

### 11.5 删除菜单

- `DELETE /menus/{id}` → `{ id: number }`

（级联策略见 §14。）

### 11.6 移动菜单（同级排序 / 调整父级）

## 12. 页面编辑器接口

### 12.1 节点树读取

#### 版本

- `GET /page-versions/{versionId}/node-tree`

返回：

```ts
{
  page_version: PageVersion
  nodes: PageNode[]
}
```

#### 模板

- `GET /page-templates/{id}/node-tree`

返回：`{ template: PageTemplate, nodes: PageNode[] }`

#### 片段

- `GET /reusable-fragments/{id}/node-tree`

返回：`{ fragment: ReusableFragment, nodes: PageNode[] }`

### 12.2 整棵树保存

- `PUT /page-versions/{versionId}/node-tree`
- `PUT /page-templates/{id}/node-tree`
- `PUT /reusable-fragments/{id}/node-tree`

请求体：

```ts
{
  nodes: PageNodeTreeWriteItem[]
}
```

返回：`{ nodes: PageNode[] }`

要求：

- **全量覆盖**该上下文下的节点集合
- 返回持久化后的完整平铺列表
- 新建节点 `id` 可为 `null`；临时负 ID 映射规则见 §14

### 12.3 新增节点（页面版本上下文）

- `POST /page-versions/{versionId}/nodes`

请求体：**CreatePageNodeInput**  
返回：**PageNode**（正式 `id`）

### 12.4 更新节点

- `PUT /page-nodes/{id}`

请求体：**UpdatePageNodeInput**（不得含 `parent_id`、`slot_name`、`sort_order`）  
返回：**PageNode**

### 12.5 移动节点

- `POST /page-nodes/{id}/move`

请求体：`{ parent_id: number | null, slot_name: string | null, sort_order: number | null }`  
返回：**PageNode**

### 12.6 删除节点

- `DELETE /page-nodes/{id}` → `{ id: number }`

### 12.7 复制节点子树

- `POST /page-nodes/{id}/copy`

可选请求体：`{ parent_id?: number | null, slot_name?: string | null, sort_order?: number | null }`

返回：`{ nodes: PageNode[] }`

### 12.8 `fragment_ref` 节点

- `node_type = 'fragment_ref'`
- `ref_fragment_id` 必填
- `component_key` 可为空
- 片段列表数据来源：`GET /reusable-fragments`

## 13. 可不强制要求的辅助能力

以下可由前端本地处理或沿用列表接口，**后端可不单独提供**：

1. slot 候选项（前端合并默认 slot 与树内 slot）
2. 父节点候选项（由当前树推算）
3. 通用下拉：页面 / 组件 / 数据绑定 / 片段均用对应列表接口

**可选用**（前端草案中存在、非库表要求）：`POST /page-nodes/{id}/reference-fragment` 将节点设为片段引用（等价于更新节点内容字段亦可）。
