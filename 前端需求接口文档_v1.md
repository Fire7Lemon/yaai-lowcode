# 前端需求接口文档

## 1. 文档说明

本文档用于明确当前前端已确定的接口需求，供后端实现与前后端联调用。

本文档仅描述当前前端已经实际使用或已明确需要的接口能力，不包含未进入当前实现范围的扩展设计。编辑器相关接口以现有前端实现为准，重点明确节点树读取、节点结构调整、节点内容更新及整树保存的接口边界。

------

## 2. 统一约定

### 2.1 基本约定

- 接口风格：REST 为主
- 主键类型：`number`
- 布尔状态字段：`boolean`
- 时间字段：ISO 8601 字符串
- JSON 配置字段：前端以字符串传输，内容为 JSON 文本

上述约定与当前前端类型和表单实现保持一致。

### 2.2 统一返回结构

列表接口返回结构：

```ts
{
  items: T[],
  total: number,
  page?: number,
  page_size?: number
}
```

详情接口返回结构：

```ts
T
```

删除接口返回结构：

```ts
{
  id: number
}
```

节点树保存接口返回结构：

```ts
{
  nodes: PageNode[]
}
```

当前前端多数列表页主要使用 `items` 与 `total`，但后端接口仍需保留分页能力。

### 2.3 通用约束

- 列表接口统一支持筛选参数
- 节点树相关接口统一返回**平铺数组 `nodes[]`**
- 编辑器结构字段与内容字段分离：
  - 结构字段：`parent_id`、`slot_name`、`sort_order`
  - 内容字段：标题、组件 key、绑定、`props_json`、`style_json` 等
- 节点结构调整通过移动接口完成，不通过节点更新接口完成

以上约束已在当前前端编辑器实现中形成明确依赖。

## 12. 无需单独提供的辅助接口

当前前端不要求后端额外提供以下专用接口：

1. slot 候选项接口
   前端本地维护默认 slot，并合并树内已有 slot
2. 父节点候选项接口
   前端基于当前树内容器节点自行计算
3. 专用下拉接口
   页面、组件定义、数据绑定、可复用片段的下拉数据均直接使用各自列表接口

这些结论已在当前前端实现中固定。

## 13. 需后端确认

1. 列表分页参数是否统一使用 `page` / `page_size`
2. 列表筛选字段采用模糊匹配还是精确匹配
3. 列表统一返回结构是否固定为 `{ items, total, page, page_size }`
4. 节点树接口统一返回平铺 `nodes[]`
5. 节点更新接口与节点移动接口的职责边界按本文执行，不混用
6. 整树保存时，新节点临时 ID 到正式 ID 的映射规则
