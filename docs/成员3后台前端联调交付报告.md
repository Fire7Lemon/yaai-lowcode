## 3. 关键技术处理

- **API 适配层**：后端 camelCase ↔ 前端类型 snake_case；不在类型层把 JSON 配置改成 object。  
- **真实模式**：**不 mock fallback** 掩盖 **`success:false`**。  
- **JSON 字段**：保持 **`string | null`**；属性面板 patch 前对非空字符串做 **`JSON.parse`** 校验。  
- **node-tree**：保存体 **`{ nodes: [...] }`** **平铺**；**`buildSaveNodeTreeRequestBodyForBackend`** 统一 camelCase。  
- **职责分离**：结构变更在页面版本走 **move**；**update** 不传结构字段。  
- **保存后 reload**：**`replaceNodesFromResponse`** 后再 **`loadEditor`**，对齐 **`id_map`** 与服务端树。  
- **用户体验**：**ElMessage** 失败提示；删除 / 保存失败 **不假成功**。
