# 后台前端功能需求

## 一、项目定位

本项目是一个基于 Vue 3 的平台型低代码后台前端。

本项目的目标不是做传统的固定栏目 CMS 后台，也不是单纯的增删改查管理系统，而是建设一个围绕以下核心模型展开的页面搭建后台：

- `page`
- `page_version`
- `page_template`
- `reusable_fragment`
- `component_def`
- `data_binding`
- `page_node`
- `menu`

本项目的核心编辑链路为：

**页面（page） -> 页面版本（page_version） -> 页面节点树（page_node）**

后台前端必须围绕这条主线实现页面管理、结构编辑、组件配置、数据绑定、模板复用和预览能力。

---

## 二、当前建设目标

当前阶段的后台前端，需要先完成一个“可用的低代码页面搭建后台基础版”。

重点目标不是一次性做完整产品，而是先完成以下核心能力：

1. 页面管理
2. 页面版本管理
3. 页面模板管理
4. 可复用片段管理
5. 菜单管理
6. 组件定义查看与配置
7. 数据绑定管理

## 3.9 页面编辑器模块（核心）

### 15）页面编辑器页
这是整个后台前端的核心页面。

页面编辑器必须围绕 `page_node` 实现，整体布局为：

- 左侧：节点树
- 中间：预览区
- 右侧：属性面板

#### 左侧节点树功能
- 展示当前 `page_version` 下的 `page_node` 树
- 支持展开 / 收起
- 支持选中节点
- 支持新增子节点
- 支持删除节点
- 支持复制节点
- 支持排序
- 支持查看 `slot_name`
- 支持区分容器节点、普通组件节点、片段引用节点

#### 中间预览区功能
- 根据节点树递归渲染页面结构
- 能看出容器和普通组件层级
- 选中节点时高亮预览对应区域
- 支持基础占位渲染
- 数据未接通时允许 mock 占位
- 必须仍然服从数据库字段，不得自造新结构

#### 右侧属性面板功能
- 根据当前选中的节点展示可编辑属性
- 必须围绕 `page_node` 字段展开
- 属性面板建议分区：
    - 基础信息
    - 布局配置
    - 业务属性
    - 样式配置
    - 数据绑定
    - 事件配置
    - 显示规则

可编辑字段至少包括：

- `node_name`
- `node_type`
- `component_key`
- `slot_name`
- `sort_order`
- `depth`
- `col_span`
- `row_span`
- `data_binding_id`
- `ref_fragment_id`
- `props_json`
- `style_json`
- `layout_json`
- `event_json`
- `visible_rule_json`
- `status`
- `remark`

## 四、页面编辑器中的节点类型要求

节点至少需要支持以下三类：

1. `container`
2. `component`
3. `fragment_ref`

说明：

- `container`：容器节点，可继续挂子节点
- `component`：普通组件节点，通常为叶子节点
- `fragment_ref`：片段引用节点，引用 `reusable_fragment`

## 七、当前阶段不强制一次性做完的能力

以下能力可以先预留结构，不要求第一版全部做完：

- 节点拖拽排序的高级交互
- schema 驱动的完整动态表单引擎
- 复杂事件编排
- 多人协同
- 版本对比
- 发布审批流
- 高级权限控制

第一版先保证：

- 数据模型正确
- 页面结构正确
- 节点树正确
- 编辑链路正确
- 预览链路正确

## 九、最终要求

整个后台前端必须始终严格围绕当前数据库设计展开。

不允许为了图省事而自行创造新的字段、新的页面结构、新的组件模型、新的数据绑定模型。

本项目追求的是：

**严格贴合数据库的平台型低代码后台前端。**
