# YAAI 开发指南（团队分工与仓库约定）

> 注：原仓库中曾分散在 **`后台前端开发范围说明`** 等文档中的范围描述已归档至 **`docs/archive/`**；本文件作为 **成员 3 / 成员 4** Pull 后的统一入口说明。

## 1. 仓库形态

- **本仓库（`yaai-lowcode-0423`）**：当前主要为 **Vue 3 + Vite + TypeScript + Element Plus** 的 **后台管理前端**（低代码搭建后台），并已接入真实 HTTP 模式。
- **前台官网**：可由 **成员 4** 在同一 monorepo 新目录或独立仓库实现；消费后台产出的 **菜单 / 页面 / 版本 / 节点树 / 组件定义 / 数据绑定** 等配置（见 **`docs/前台接入后台低代码说明.md`**）。

## 2. 成员 3（后台前端）

**职责：**

- 页面管理、菜单管理、组件定义、数据绑定、页面模板、可复用片段、页面版本。
- 低代码 **页面节点树编辑器**（`page_version` / `page_template` / `reusable_fragment` 三场景）。
- **`src/api/*`** 与后端字段风格差异的适配（camelCase ↔ 类型层 snake_case）。
- 真实模式下 **不假成功、不 mock 掩盖接口失败**。

**必读：**

- 根目录 **`AGENTS.md`**（Cursor Agent 硬性约束，与数据模型一致）。
- **`docs/文档索引.md`**、**`docs/联调总进度.md`**、**`docs/前端联调修复报告.md`**。
- 交付与后端对接：**`docs/成员3后台前端联调交付报告.md`**、**`docs/后端接口修复要求_交付版.md`**。

## 3. 成员 4（前台前端）

**职责：**

- **官网前台展示**：菜单导航、按路由展示页面、拉取 **published（或约定状态）** 的页面版本与 **node-tree**，将 **平铺 nodes** 转成树并 **按 `component_key` 映射渲染**。
- **不做**：后台管理 UI、低代码编辑器、对 **`page_node`** 的增删改移动与整树保存。

**必读顺序：** 见 **`docs/文档索引.md` →「成员 4 优先阅读」**，以及 **`docs/前台接入后台低代码说明.md`**。

## 5. 环境与路径提醒

- 浏览器开发：**`/api/...`**（`VITE_API_BASE_URL`）。
- **curl 直连后端**：`http://127.0.0.1:9876/...`，**不要**多一层 **`/api`**（详见 **`docs/联调总进度.md`** 阶段 E-6.1）。
