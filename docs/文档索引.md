## 4. 路径约定（联调结论）

| 场景 | 约定 |
|------|------|
| **浏览器 / Vite** | 请求前缀 **`/api/...`**（与 `VITE_API_BASE_URL` 一致） |
| **curl 直连后端** | **`http://127.0.0.1:9876/...`**，**不要**在端口后再加 **`/api`**（误用易出现 `SYSTEM_ERROR`，详见 **`docs/联调总进度.md`**） |

## 5. 重要约束（全员）

1. **不要擅自修改**数据库设计正文（**`docs/数据库设计_v6.0_平台型低代码版.md`**）。
2. **不要擅自修改**接口契约正文（**`docs/前端需求接口文档_v2.0.md`**）；运行差异记在 **`docs/接口字段差异记录.md`**。
3. **前端类型与表单层**保持 **`snake_case`** 语义（对齐数据库字段名）。
4. **后端真实读写**常见 **camelCase**，由 **`src/api/*`** 适配（成员 4 若写前台 API 层，遵循同一原则）。
5. **JSON 配置字段**保持 **`string | null`**，渲染时再 **`JSON.parse`**，并做好兜底。
6. **真实模式**不允许用 mock **掩盖**接口失败。
7. **后台低代码**负责编辑与保存配置；**前台**负责读取配置并渲染，不负责节点树编辑落库。
