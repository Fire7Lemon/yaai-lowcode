# 接口字段差异记录

说明：文档字段以 snake_case 为准；若后端请求/响应为 camelCase，仅在 `src/api/*` 做适配。此处记录已观测或已适配的差异，**不擅自修改接口契约文档**。

## 模块写入请求 — camelCase 适配汇总（阶段 G）

以下模块 **POST/PUT**（及 **page-node** 单节点写入）在真实后端环境下 **`src/api/*`** 均以 **camelCase** 请求体写出；**前端 TS 类型与表单字段仍为 snake_case**；**JSON 配置字段（如 `propsJson`、`queryJson`）仍为 `string | null`**。

| 模块 | 典型接口 | API 适配要点 |
|------|----------|----------------|
| **page** | `POST/PUT /pages` | **`buildPageWriteRequestBodyForBackend`** 等（camelCase） |
| **menu** | `POST/PUT /menus` | 写入 camelCase |
| **component-def** | `POST/PUT /component-defs` | 写入 camelCase |
| **data-binding** | `POST/PUT /data-bindings` | 写入 camelCase |
| **page-template** | `POST/PUT /page-templates` | **`buildPageTemplateWriteRequestBodyForBackend`** |
| **reusable-fragment** | `POST/PUT /reusable-fragments` | 与模板对称 |
| **page-version** | 列表创建等 | **`mapPageVersion`** 读响应 |
| **page-node** | `POST .../nodes`、`PUT /page-nodes/{id}`、**move** | **`buildCreatePageNodeRequestBody`**、**`buildUpdatePageNodeBodyCamel`** 等 |
| **node-tree（三类）** | **`PUT .../node-tree`** | **`buildSaveNodeTreeRequestBodyForBackend`**，**`{ nodes }`** 平铺 |

## 浏览器请求路径 vs 直连后端路径（E-6.1）

| 场景 | 典型 URL / 前缀 | 说明 |
|------|-----------------|------|
| **浏览器 / Vite 代理** | **`/api/page-templates`**、**`/api/reusable-fragments`** 等 | 与 **`VITE_API_BASE_URL`** 一致；代理将请求转发到后端实际监听路径 |
| **curl 直连 `127.0.0.1:9876`** | **`http://127.0.0.1:9876/page-templates`**、**`http://127.0.0.1:9876/reusable-fragments`** | **不要** 在主机端口后再加 **`/api`**；本机误用 **`/api/...`** 会得到 **`SYSTEM_ERROR`**，易误判为列表接口故障 |

## 浏览器 Network — 模板 / 片段 node-tree PUT（E-7）

- **路径**：**`PUT /api/page-templates/{id}/node-tree`**、**`PUT /api/reusable-fragments/{id}/node-tree`**（**Vite** **`/api`** 前缀；代理转发至后端业务路由）。
- **请求体**：**`{ "nodes": [...] }`**，**平铺**；节点字段 **camelCase**（如 **`templateId`、`fragmentId`、`parentId`、`nodeType`、`propsJson`**），来自 **`buildSaveNodeTreeRequestBodyForBackend`**。
- **前端内部**：**`SavePageNodeTreePayload`** / **`PageNodeTreeWriteItem`** 仍为 **snake_case**；仅在 **`src/api/page-template.ts`**、**`src/api/reusable-fragment.ts`** 写出时映射。

## page-version — node-tree（GET）

- **模块**：page-version / page-node
- **接口**：`GET /page-versions/{versionId}/node-tree`
- **文档字段**：期望 `data` 内含 `nodes` 平铺数组及版本元数据（命名以文档为准）
- **后端实际字段**（本机样例 `versionId=307`）：
  - 信封：`code` / `success` / `message` / `data`
  - `data.nodes[]`：每项为 **camelCase**（如 `pageVersionId`、`nodeType`、`componentKey`、`propsJson`、`slotName`、`parentId`、`sortOrder`、`dataBindingId`、`refFragmentId`）
  - `data.page_version`：版本摘要对象（键为 **snake_case `page_version`**，对象内字段仍为 **camelCase**，如 `pageId`、`versionNo`）
- **前端适配方式**：`src/api/page-node.ts` 中 `mapPageNode`、`mapPageVersion` 同时读取 snake_case 与 camelCase；版本树读取使用 `extractEnvelopeData`，从 `data.nodes` 取平铺列表
- **是否需后端统一**：若文档规定响应全 snake_case，建议后端统一；当前以前端映射为主

## page-node — 单节点写入（POST/PUT/MOVE/DELETE）

- **模块**：page-node
- **接口**：
  - `POST /page-versions/{versionId}/nodes`（创建）
  - `PUT /page-nodes/{id}`（更新）
  - `POST /page-nodes/{id}/move`（结构）
  - `DELETE /page-nodes/{id}`（删除）
- **文档字段**：前端类型与表单仍为 **snake_case**（`CreatePageNodeInput`、`UpdatePageNodeInput`、`MovePageNodeInput`）
- **后端实际字段（写）**：本机实测请求体为 **camelCase**（如 `parentId`、`nodeType`、`componentKey`、`propsJson`、`slotName`、`sortOrder`）
- **前端适配方式**：仅在 **`src/api/page-node.ts`** 内 **`buildCreatePageNodeRequestBody`** / **`buildUpdatePageNodeBodyCamel`** / move 的 JSON 体做 camelCase；**不在类型层改为 object**
- **update 约束**：PUT 体 **不包含** `parentId` / `slotName` / `sortOrder`（结构变更 **仅** 走 move）
- **move 约束**：POST move 体 **仅** `parentId`、`slotName`、`sortOrder`
- **JSON 字段**：请求与响应中 `propsJson` 等仍为 **字符串或 null**（与 `string | null` 一致）
- **DELETE 响应**：本机实测 **`data` 为数字**（节点 id）；适配层兼容 `{ id }` 与缺省回退请求 id
- **copy 响应（观测）**：副本节点可能出现 **`parentId: null`**（见 **`docs/后端待修复清单.md` §7**）
- **是否需后端统一**：若文档规定请求 snake_case，建议后端双风格兼容或统一并在文档明示

## node-tree — 整树保存 PUT（三类路径统一 camelCase 写出）

以下三条接口在本机对 **页面版本** 路径已验证：**snake_case** 请求体可导致 **`success:true`** 但 **`parentId`** 丢失、树结构破坏（**`docs/后端待修复清单.md` §8**）。模板 / 片段此前代码同为 snake_case **`JSON.stringify(payload)`**，**E-6** 已与版本路径对齐为同一构造函数。

### 浏览器 Network（Vite + `/api`）

- **路径**：开发环境下请求形如 **`PUT /api/page-templates/{id}/node-tree`**、**`PUT /api/reusable-fragments/{id}/node-tree`**（由代理转发至后端根路径；与 **E-6.1** curl **直连无 `/api`** 对照）。
- **请求体**：**`{ "nodes": [...] }`**，数组 **平铺**；节点字段 **camelCase**（如 **`templateId`、`fragmentId`、`parentId`、`nodeType`、`propsJson`**），与 **`buildSaveNodeTreeRequestBodyForBackend`** 一致。
- **前端内部**：**`editorStore`** / **`buildSavePageNodeTreePayload`** 仍为 **`PageNodeTreeWriteItem`** **snake_case**，仅在 **`src/api/*`** 映射后写出。

### page-version

- **接口**：`PUT /page-versions/{versionId}/node-tree`
- **前端调用**：**`saveVersionNodeTree`**（**`src/api/page-node.ts`**）

### page-template

- **接口**：`PUT /page-templates/{id}/node-tree`
- **前端调用**：**`saveTemplateNodeTree`**（**`src/api/page-template.ts`**）

### reusable-fragment

- **接口**：`PUT /reusable-fragments/{id}/node-tree`
- **前端调用**：**`saveFragmentNodeTree`**（**`src/api/reusable-fragment.ts`**）

### 共用适配

- **文档字段**：前端内部 **`SavePageNodeTreePayload`**（**`PageNodeTreeWriteItem`**，**snake_case**）不变
- **后端实际字段（写）**：须 **`camelCase`** 平铺 **`{ nodes: [...] }`**（如 **`pageVersionId` / `templateId` / `fragmentId`、`parentId`、`propsJson`** 等）；**不要**提交嵌套树
- **前端适配方式（E-5～E-6）**：**`shouldUseRealApi()`** 下 **`JSON.stringify(buildSaveNodeTreeRequestBodyForBackend(payload))`**（定义于 **`src/api/page-node.ts`**，导出供模板/片段 API 复用）
- **响应**：成功时常见 **`data.nodes`**；版本路径另见 **`data.id_map`**（**§9**）
- **JSON 字段**：请求与响应保持 **字符串或 null**，适配层 **不 parse** 写入体中的 JSON 字段
- **模板 / 片段 PUT 实测（E-6.2）**：本机 **`127.0.0.1:9876`** **curl**（**无 `/api`**）：专用资源 **POST** → **GET** **`nodes`**（空数组）→ **camelCase** **`PUT /page-templates/{id}/node-tree`** / **`PUT /reusable-fragments/{id}/node-tree`**（**`{ nodes: [...] }`**，单根 **`id: null`**，JSON 字段字符串或 **`null`**）→ **GET** 校验节点数与 **`parentId`/`templateId`/`fragmentId`** → **DELETE** 清理。**模板节点 id** **`4106`**、**片段节点 id** **`4107`**（随 **`DELETE` 模板/片段** 一并清理；资源 **id=4** 已删除，**无遗留**）。与 **`buildSaveNodeTreeRequestBodyForBackend`** 写出风格一致。

## editor store（页面版本）

- **模块**：低代码编辑器 Pinia store
- **接口**：不直接对应单一 HTTP；通过 **`src/api/page-node.ts`** 调用后端
- **文档字段**：store 与表单仍使用 **`CreatePageNodeInput` / `UpdatePageNodeInput` / `MovePageNodeInput`** 的 **snake_case** 语义
- **后端实际字段**：写请求由 **`page-node.ts`** / **`page-template.ts`** / **`reusable-fragment.ts`** 转为 **camelCase**（含三类 **`save*NodeTree`** 整树 PUT，**E-5～E-6**）
- **前端适配方式**：**`pickContentUpdatePayload`** 仅传递允许的内容字段；**`moveNode`** 仅向 **`movePageNode`** 传递 **`parent_id` / `slot_name` / `sort_order`**
- **职责分离**：**update** 不提交结构字段；结构变更仅 **move** 接口（页面版本真实模式）
- **是否需后端统一**：否（适配层已收敛）
