# 前台接入后台低代码说明

面向 **成员 4（官网前台前端）**：Pull 本仓库或对接同一后端时，如何用后台产出的低代码配置渲染官网。

---

## 1. 成员 4 职责边界

- **负责**：官网前台 **展示系统**（导航、路由页、按配置渲染内容）。
- **不负责**：本仓库中的 **后台管理系统** UI、**低代码编辑器**。
- **不负责**：对 **`page_node`** 的 **新增 / 删除 / 移动 / 复制** 或 **整树 PUT**（均由后台完成）。
- **负责**：**读取**已发布的菜单、页面、版本与节点树，并 **稳定渲染**。

---

## 2. 前后台关系

### 2.1 后台低代码生产的数据与配置

| 实体 | 作用 |
|------|------|
| **menu** | 导航结构、指向页面或外链 |
| **page** | 页面元数据、**path**、**current_version_id**（或等价字段） |
| **page_version** | 某页的某一版内容快照入口 |
| **page_node** | 页面结构树节点（平铺存储，前台需还原为树） |
| **component_def** | **`component_key`** 与 schema / 默认 props 等 |
| **data_binding** | 列表/区块类组件的数据源配置 |
| **reusable_fragment** | 可复用片段（节点树中可能以片段引用等形式出现） |

### 2.2 前台消费方式（概念）

- 拉菜单 → 渲染导航栏。
- 用户进入 **`page.path`** → 解析当前应展示的 **版本 id**（常与 **`page.currentVersionId`** 或发布链路约定一致，以后端与契约为准）。
- 拉 **node-tree** → **flat nodes → tree** → 递归 **`NodeRenderer`**。
- 叶子/容器节点根据 **`component_key`** 映射到 Vue 组件；结合 **`data_binding_id`** 等拉取展示数据（按业务接口约定）。

## 3. 前台最小数据流（推荐步骤）

以下为逻辑顺序；具体路径以 **`docs/前端需求接口文档_v2.0.md`** 及 **`docs/接口字段差异记录.md`** 为准。

1. **`GET /menus`**（或契约约定的菜单列表接口）  
2. **组装菜单树**（根据 **`parentId`** 等字段）。  
3. 用户点击菜单 → 进入 **`page.path`**（前端路由与后端 **`path`** 对齐）。  
4. **`GET /pages`** 或 **`GET /pages/{id}`** / 后续若有的 **public page-by-path** 接口 → 得到 **`page`**。  
5. 读取 **`page.currentVersionId`**（或发布版本解析规则）。  
6. **`GET /page-versions/{versionId}/node-tree`** → 得到 **平铺 `nodes`**（响应多为 **camelCase**，前台适配层映射为自己的类型）。  
7. **flat nodes → tree**（按 **`parentId`**、**`slotName`**、**`sortOrder`** 等）。  
8. **`NodeRenderer`** 递归渲染；未知 **`component_key`** 走兜底组件，避免整页崩溃。

## 4. 前台推荐目录结构（示例）

成员 4 可在 **`src/`** 下新建独立命名空间（勿与后台 **`views/`** 混用），例如：

```text
src/
  api/
    public-menu.ts
    public-page.ts
    public-node-tree.ts

  components/
    layout/
      PublicHeader.vue
      PublicFooter.vue
      PublicLayout.vue

    renderer/
      DynamicPage.vue
      NodeRenderer.vue
      component-map.ts

    website/
      HeroBanner.vue
      NoticeList.vue
      NewsList.vue
      QuickLinks.vue
      RichTextBlock.vue
      ContainerBlock.vue
      UnknownComponent.vue

  views/
    DynamicPageView.vue
    NotFoundView.vue
```

实际文件名可与团队约定一致；关键是 **API / 渲染器 / 组件映射** 分层清晰。

## 5. component_key 映射

**`component_key`** 是后台配置与前台 Vue 组件之间的桥。

| component_key（示例） | 前台组件（示例） |
|----------------------|------------------|
| hero_banner | HeroBanner |
| notice_list | NoticeList |
| news_list | NewsList |
| quick_links | QuickLinks |
| rich_text | RichTextBlock |
| container | ContainerBlock |
| nav_menu | NavMenuBlock |
| （未知） | UnknownComponent |

**要求：**

- **`component-map.ts`** 集中维护映射。
- **未知 `component_key`**：不得导致应用崩溃；渲染 **`UnknownComponent`**（可打日志、降级 UI）。

---

## 6. JSON 字段处理

- **`props_json` / `style_json` / `layout_json` / `event_json` / `visible_rule_json`** 在模型中为 **`string | null`**。
- **渲染时**再 **`JSON.parse`**；失败则使用 **安全兜底**（空对象或默认样式）。
- **不要把数据库字段改成 object 写入契约层类型**（类型语义仍以 v6.0 / 后台类型为准）。
- **单个节点 JSON 错误**不应拖垮整页（捕获并降级）。

---

## 7. 前台不要做的事

1. 不做后台低代码编辑器。  
2. 不 **新增 / 删除 / 移动** **`page_node`**（结构变更在后台完成）。  
3. 不调用 **整树保存** **`PUT …/node-tree`**（除非产品明确要求前台具备「预览草稿提交」且后端单独约定——默认不做）。  
4. **不修改**数据库设计与接口契约正文。  
5. **不依赖** mock 假数据冒充线上接口成功。  
6. **不写死**全部官网页面结构（应以配置驱动）。  
7. **不绕过后台配置**硬编码与 **`component_key`** 无关的页面大块。

## 8. 成员 4 第一阶段任务建议（小步）

1. 扫描当前仓库是否已有前台目录；若无则建立 **`src/api/public-*`** 与 **`renderer`** 骨架。  
2. 实现 **`public-menu`** API + **导航栏**。  
3. 实现 **`DynamicPageView`** + 路由与 **`path`** 对齐。  
4. 实现 **`getPageNodeTree`/`public-node-tree`**。  
5. 实现 **flat → tree** 工具函数（可与后台 **`buildNodeTree`** 思路对齐，但不要复制后台编辑器逻辑）。  
6. 实现 **`NodeRenderer`** + **`component-map`**。  
7. 先打通 **首页（或单页）动态渲染闭环**，再扩展绑定数据与更多 **`component_key`**。
