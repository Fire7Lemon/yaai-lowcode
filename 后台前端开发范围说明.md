# 后台前端开发范围说明

## 一、项目定位

本项目是一个从 0 开始搭建的平台型低代码后台前端。

本项目的目标不是传统 CMS 后台，也不是单纯的表单管理后台，而是一个围绕页面结构树、组件注册、数据绑定、模板与复用片段展开的页面搭建后台。

## 二、当前开发角色

当前仅建设后台前端。

重点服务于以下数据模型：

- page
- page_version
- page_template
- reusable_fragment
- component_def
- data_binding
- page_node
- menu

## 三、当前前端建设范围

本阶段需要完成的主要页面与功能包括：

1. 页面管理
2. 页面版本管理
3. 页面模板管理
4. 可复用片段管理
5. 菜单管理
6. 组件定义查看与配置
7. 数据绑定管理
8. 页面节点树编辑器
9. 页面预览区
10. 右侧属性面板

## 四、当前不作为重点建设的内容

当前不主动扩展以下模块：

- 用户系统
- 权限系统
- 复杂审计系统
- 与数据库设计无关的外围业务系统

## 五、页面编辑器核心原则

页面编辑器必须以 `page_node` 作为唯一结构核心。

编辑器整体形态为：

- 左侧：节点树
- 中间：预览区
- 右侧：属性面板

其中：
- 节点树来源于 `page_node`
- 组件能力来源于 `component_def`
- 数据来源配置来源于 `data_binding`
- 页面版本切换来源于 `page_version`

## 六、开发原则

所有实现必须严格贴合当前数据库设计，不允许脱离数据库自由发挥。
